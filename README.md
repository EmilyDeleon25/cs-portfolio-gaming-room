# CS Portfolio – The Gaming Room Software Design

## Client Summary
The client for this project was The Gaming Room. They wanted to expand their game, Draw It or Lose It, into a web-based application that could support multiple teams and players. 
The goal was to design a system that could run across platforms while maintaining performance, scalability, and security.

## What I Did Well
One thing I did well in this project was explaining the system architecture and recommending technologies that would support scalability and reliability for the client.

## How the Design Document Helped
Creating the design document helped organize the development process before writing any code. It made it easier to understand how the system should work and how different components would interact.

## What I Would Improve
If I could revise one section, I would improve the system architecture portion by adding more detailed explanations and diagrams to make the design easier for developers to follow.

## Understanding User Needs
I focused on the client’s goal of expanding their game to support multiple users and platforms. Considering user needs is important because the software must solve the problem the client is trying to address.

## My Approach to Software Design
My approach was to first analyze the client’s requirements and then research technologies that would best support those needs. 
In the future, I would continue using this structured approach to analyze and design similar applications.
